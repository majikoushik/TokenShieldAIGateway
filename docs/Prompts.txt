# TokenShield AI Gateway — Epic-Based Codex Prompt Pack

Use these prompts after placing the updated `AGENTS.md` in the repository root and, ideally, in your coding tool's global project rules.

These prompts intentionally avoid repeating the full architecture and security policy. Codex must treat `AGENTS.md` as the source of truth.

Recommended use:

1. Run one epic prompt at a time.
2. Let Codex inspect the repository before editing.
3. Ask Codex to commit or summarize changes after each epic.
4. Do not move to the next epic until build/test issues are resolved.
5. Use the final verification prompt after all epics are complete.

## 2. Feature Flag / Configuration

Add configuration:

```json
{
  "RequestProfiler": {
    "Mode": "Mvp",
    "EnableProductionProfiler": false,
    "EnableSemanticClassifier": false,
    "EnableLlmClassifier": false,
    "LowConfidenceThreshold": 0.6,
    "HighConfidenceThreshold": 0.85,
    "DefaultTaskType": "general",
    "DefaultRiskLevel": "medium",
    "UseMetadataAsPrimarySignal": true,
    "ValidateMetadataAgainstPrompt": true,
    "EnableSensitivityDetection": true,
    "EnableProfileSignals": true
  }
}
```

Supported modes:

```text
Mvp
Production
Hybrid
```

Behavior:

1. Mode = Mvp

   * Use the existing MVP profiler exactly as before.

2. Mode = Production

   * Use new production profiler.

3. Mode = Hybrid

   * Run production profiler first.
   * If production confidence is too low, fall back to MVP profiler or safe default routing metadata.

Feature flag behavior:

* If EnableProductionProfiler=false, use MVP profiler.
* If EnableProductionProfiler=true and Mode=Production, use production profiler.
* If EnableProductionProfiler=true and Mode=Hybrid, use hybrid profiler.

Default:

* Keep default as MVP mode to avoid breaking existing behavior.

Environment variable examples:

```text
RequestProfiler__Mode=Production
RequestProfiler__EnableProductionProfiler=true
RequestProfiler__EnableSemanticClassifier=false
RequestProfiler__EnableLlmClassifier=false
RequestProfiler__LowConfidenceThreshold=0.6
RequestProfiler__HighConfidenceThreshold=0.85
```

## 12. Confidence and Safe Defaults

Production profiler must not pretend certainty.

Rules:

1. If taskType confidence < LowConfidenceThreshold:

   * taskType = general
   * add warning
   * classificationMethod should include low_confidence
2. If risk confidence < LowConfidenceThreshold:

   * riskLevel = medium
   * add warning
3. If task confidence is high but risk is uncertain:

   * keep task type
   * default risk to medium
4. If sensitivity is uncertain:

   * dataSensitivity = unknown
5. If high-risk signal exists:

   * do not downgrade risk because of metadata.

This makes routing safer.

## 13. Hybrid Profiler

Implement HybridRequestProfiler.

Behavior:

1. Run ProductionRequestProfiler.
2. If production task confidence >= LowConfidenceThreshold:

   * use production profile.
3. If production task confidence < LowConfidenceThreshold:

   * run MVP profiler.
4. Merge:

   * keep production risk and sensitivity if stronger.
   * use MVP task type only as fallback.
   * classificationMethod = hybrid.
5. Add warning:

   * "Production profiler confidence was low; MVP fallback was used."

Hybrid mode gives safety during rollout.
